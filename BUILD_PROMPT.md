# LogosLight - Complete Build Specification

**For sharing with development platforms or external teams**

---

## Project Overview

**LogosLight** is a free Christian devotional web app that combines KJV Scripture with Claude Haiku AI to provide personalized spiritual guidance. 

**Tagline:** *"God's Word for every moment of your day."*

---

## Core Features

### 1. Verse of the Day
- Displays a random KJV Bible verse on the home page with each visit
- Includes reflection and daily application prompts
- Fresh verse every page load

### 2. Emotion-Based Devotional
Users select or describe an emotion, receive:
- Relevant KJV scripture passages (curated + AI-enhanced)
- AI-generated prayer tailored to their emotion
- Personalized encouragement & explanation
- Supports both preset emotions and custom free-text input

### 3. AI Deep Study Guide
Users select or input a topic, receive:
- Scripture-based study guide
- Sermon preparation notes
- Worship planning suggestions
- Theological exploration for any Biblical topic

## Architecture

### High-Level Flow

```
User Browser (React SPA on Vercel)
    ↓
FastAPI Backend (Render.com)
    ├→ Supabase DB (verse lookups)
    └→ Claude Haiku API (text generation)
```

### Data Flow

**Verse of Day:**
```
GET /api/verses/verse-of-day
→ DB: SELECT * FROM verses ORDER BY RANDOM() LIMIT 1
→ Return verse + reflection
```

**Emotion Devotional:**
```
POST /api/devotional/emotion { emotion: "string" }
→ Look up relevant verses from VERSE_MAP[emotion]
→ Fetch verse texts from Supabase
→ Call Claude Haiku with verses + emotion
→ Return prayer + encouragement + full response
```

**Deep Study:**
```
POST /api/ai-study/search { topic: "string" }
→ Look up relevant verses from STUDY_MAP[topic]
→ Fetch verse texts from Supabase
→ Call Claude Haiku with verses + topic
→ Return study guide + sermon notes + application
```

## Database Schema

### Single Table: `verses`

```sql
CREATE TABLE verses (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    book         VARCHAR(50)  NOT NULL,
    chapter      INT          NOT NULL,
    verse_number INT          NOT NULL,
    text         TEXT         NOT NULL,
    version      VARCHAR(10)  DEFAULT 'KJV',
    created_at   TIMESTAMP    DEFAULT NOW(),
    UNIQUE (book, chapter, verse_number)
);

CREATE INDEX idx_verses_ref ON verses(book, chapter, verse_number);

CREATE OR REPLACE FUNCTION get_random_verse()
RETURNS TABLE(book VARCHAR, chapter INT, verse_number INT, text TEXT)
LANGUAGE sql AS $$
    SELECT book, chapter, verse_number, text
    FROM verses ORDER BY RANDOM() LIMIT 1;
$$;
```

**Key Points:**
- Total verses: 31,102 (KJV is public domain)
- No vector search / pgvector extension needed
- Standard SQL only
- Simple reference-based lookups

## API Endpoints (Backend)

### Base URL
- **Development:** `http://localhost:8000`
- **Production:** `https://logos-light-api.onrender.com`
- **API Docs:** `http://localhost:8000/docs` (auto-generated by FastAPI)

### Endpoints

#### 1. GET `/api/health`
Health check.

**Response:**
```json
{
  "status": "healthy",
  "timestamp": "2026-05-25T10:00:00"
}
```

---

#### 2. GET `/api/verses/verse-of-day`
Get a random KJV verse.

**Response:**
```json
{
  "reference": "Psalm 46:1",
  "text": "God is our refuge and strength, a very present help in trouble.",
  "book": "Psalm",
  "chapter": 46,
  "verse": 1,
  "reflection": "Meditate on this verse and how God is speaking to you today.",
  "apply_today": "Find one practical way to live out this scripture today."
}
```

---

#### 3. POST `/api/devotional/emotion`
Get emotion-based devotional content.

**Request:**
```json
{
  "emotion": "I'm anxious about my job interview tomorrow"
}
```

**Response:**
```json
{
  "emotion": "I'm anxious about my job interview tomorrow",
  "verses": [
    {
      "reference": "Philippians 4:6",
      "text": "Be careful for nothing; but in every thing by prayer and supplication with thanksgiving let your requests be made known unto God:"
    },
    {
      "reference": "Matthew 6:34",
      "text": "Take therefore no thought for the morrow: for the morrow shall take thought for the things of itself. Sufficient unto the day is the evil thereof."
    }
  ],
  "prayer": "Heavenly Father, I come before You with my anxious heart...",
  "encouragement": "God has not given you a spirit of fear, but of power, love, and a sound mind...",
  "full_response": "..."
}
```

**Notes:**
- Accepts ANY emotion string (open input)
- No hardcoded validation
- Claude Haiku handles interpretation

---

#### 4. POST `/api/ai-study/search`
Get deep study guide for any topic.

**Request:**
```json
{
  "topic": "Preparing a sermon on grace"
}
```

**Response:**
```json
{
  "topic": "Preparing a sermon on grace",
  "verses": [
    {
      "reference": "Ephesians 2:8",
      "text": "For by grace are ye saved through faith; and that not of yourselves: it is the gift of God:"
    }
  ],
  "study_guide": "Overview: Grace is...\n\nVerse Explanations: ...\n\nApplication: ...\n\nReflection Question: ..."
}
```

- `generate_emotion_devotional(emotion, verses)` function
- `generate_deep_study(topic, verses)` function

## Deployment

### Backend → Render.com

1. Create `Procfile` in backend folder:
   ```
   web: uvicorn main:app --host 0.0.0.0 --port $PORT
   ```

2. Push to GitHub

3. Connect GitHub repo to render.com → New Web Service

4. Settings:
   - Build command: `pip install -r requirements.txt`
   - Start command: `uvicorn main:app --host 0.0.0.0 --port $PORT`
   - Plan: **Free**

5. Environment variables:
   - `SUPABASE_URL`
   - `SUPABASE_KEY`
   - `ANTHROPIC_API_KEY`

### Frontend → Vercel

1. Set `VITE_API_URL` in `frontend/.env.production`:
   ```
   VITE_API_URL=https://your-render-app.onrender.com
   ```

2. Connect frontend GitHub repo to vercel.com

3. Auto-deploys on every push
